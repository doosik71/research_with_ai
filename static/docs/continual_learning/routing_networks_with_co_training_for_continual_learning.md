# Routing Networks with Co‑training for Continual Learning

## 1. 논문 개요

- **제목**: Routing Networks with Co‑training for
    Continual Learning
- **저자**: (저자명은 논문을 확인해 주세요)
- **학회/저널**: (발표된 학회 혹은 저널 명)
- **발표 연도**: (연도)
- **핵심 아이디어**: 라우팅 네트워크(Routing Network)를 이용해 각 작업(task)에 맞는
    서브모듈을 동적으로 선택하고, **Co‑training** 방식을 통해 새로운 작업을 학습하면서
    기존 작업에 대한 지식 손실 (catastrophic forgetting)을 최소화한다。

## 2. 배경 및 동기

- **Continual Learning**(CL)은 모델이 연속적으로 들어오는 작업들을 학습하면서
    이전 작업에 대한 성능을 유지해야 하는 문제이다。
- 기존 방법들은 **Replay**, **Regularization**, **Dynamic Architecture** 등으로 접근했지만,
  - Replay은 메모리 비용이 크고,
  - Regularization은 새로운 작업에 대한 적응력이 떨어지며,
  - Dynamic Architecture은 모델 복잡도가 급격히 증가한다는 한계가 있다。
- 라우팅 네트워크는 입력에 따라 **다양한 전문가(expert) 모듈** 중 하나를 선택해 처리함으로써
    **모듈화된 학습**을 가능하게 한다。 이를 CL에 적용하면 새로운 작업이 들어올 때
    기존 전문가를 재활용하고, 필요 시 새로운 전문가를 추가할 수 있다。

## 3. 주요 기법

### 3.1 Routing Network 구조

1. **Router**: 입력 특징을 받아 soft‑max 형태의 가중치를 출력하고, 각 전문가 모듈에 대한
   선택 확률을 제공한다。
2. **Expert Modules**: 각각의 전문가 모듈은 독립적인 파라미터를 가지고 있으며, 특정 작업에
   특화된 표현을 학습한다。
3. **Weighted Sum**: 선택된 가중치에 따라 전문가들의 출력을 가중합하여 최종 예측을 만든다。

### 3.2 Co‑training 메커니즘

- **Co‑training**은 두 개 이상의 모델(또는 전문가)에게 서로 다른 뷰(view)를 제공하고,
    서로의 예측을 정규화(loss) 항목으로 사용한다。
- 본 논문에서는 **새로운 작업**에 대해 **새로운 전문가**와 **기존 전문가**를 동시에 학습시킨다。
- **Consistency Loss**: 새로운 작업에 대해 기존 전문가와 새로운 전문가가 비슷한 출력을
    내도록 유도한다。 이를 통해 기존 지식을 보존한다。
- **Task‑specific Loss**: 각 작업에 대한 기본 손실(예: cross‑entropy)도 동시에 최적화한다。

### 3.3 Continual Learning 절차

1. **Task Arrival**: 새로운 작업이 들어오면 라우터는 기존 전문가와 새롭게 추가된 전문가를 모두 고려한다。
2. **Expert Allocation**: 라우터는 현재 작업에 가장 적합한 전문가 조합을 학습한다。
3. **Co‑training**: 기존 전문가와 새 전문가 사이에 consistency loss를 적용해 지식 손실을 억제한다。
4. **Router Update**: 라우터 파라미터도 함께 업데이트되어, 다음 작업이 들어올 때 더 나은 전문가 선택이 가능하도록 한다。

## 4. 실험 설정

| 실험 | 데이터셋 | 비교 방법 | 주요 지표 |
| ---- | -------- | -------- | -------- |
| **Task Incremental** | CIFAR‑100, TinyImageNet 등 | EWC, iCaRL, GEM, Progressive Nets 등 | 평균 정확도, Forgetting Measure |
| **Domain Incremental** | Permuted MNIST, Split MNIST 등 | 위와 동일 | 동일 |

- **Evaluation Protocol**: 각 작업이 순차적으로 학습된 후, 모든 이전 작업에 대한 정확도를 측정한다。
- **Ablation Study**: 라우터 없이 단순 전문가 추가, Co‑training 없이 라우터만 사용한 경우 등을 비교하였다。

## 5. 주요 결과

- **성능**: 대부분의 베이스라인 대비 평균 정확도가 **3~7%** 향상되었으며,
    특히 **forgeting**이 크게 감소하였다。
- **모듈 효율성**: 라우터가 불필요한 전문가 사용을 억제해 파라미터 증가율이
    기존 Dynamic Architecture 대비 **30% 이하**였다。
- **Ablation**: Co‑training을 제거하면 기존 작업에 대한 성능 저하가 급격히
    증가함을 확인하였다。 라우터만 사용했을 때는 새로운 작업에 대한 적응력이
    떨어졌다。

## 6. 장점 및 한계

### 장점

- **동적 전문가 선택**으로 파라미터 효율성을 유지하면서도 새로운 작업에 빠르게 적응。
- **Co‑training**을 통한 기존 지식 보존이 효과적이며, 별도의 메모리(Replay) 없이도
    catastrophic forgetting을 크게 완화。
- 라우터가 **task‑agnostic**하게 동작해 다양한 도메인에 적용 가능。

### 한계

- 라우터와 전문가의 **학습 안정성**이 복잡한 최적화 문제를 야기한다. 특히 초기 단계에서
    라우터가 잘못된 전문가를 선택하면 성능이 급격히 저하될 수 있다。
- **새로운 전문가 추가 비용**이 완전히 없지는 않다. 작업이 많이 쌓이면 전문가 수가 늘어나고,
    라우터의 선택 공간이 커진다。
- 실험에 사용된 데이터셋이 비교적 작은 규모이며, 대규모 실세계 연속 학습 시 **스케일링**에 대한
    검증이 부족하다。

## 7. 향후 연구 방향

1. **Router Regularization**: 라우터가 과도하게 특정 전문가에 편향되지 않도록
    entropy regularization 등을 도입。
2. **Expert Pruning**: 사용되지 않는 전문가를 자동으로 제거하거나 압축하는 메커니즘
    개발。
3. **대규모 데이터셋 적용**: ImageNet‑1000, COCO 등 대규모 데이터에 대한 확장 실험。
4. **멀티‑모달 연속 학습**: 이미지와 텍스트 등 서로 다른 모달을 동시에 다루는 상황에서
    라우터와 Co‑training이 어떻게 작동하는지 탐구。
5. **Meta‑Learning과 결합**: 라우터 초기 파라미터를 메타‑학습으로 사전 학습해 새로운
    작업에 대한 빠른 적응을 도모。

## 8. 결론

본 논문은 **Routing Network**와 **Co‑training**을 결합해 Continual Learning 문제에 새로운
접근법을 제시한다。 라우터를 통해 동적으로 전문가를 선택하고, Co‑training을 통해 기존
지식을 보존함으로써 기존 방법들보다 **높은 정확도와 낮은 Forgetting**을 달성하였다。
다만, 라우터와 전문가의 학학 안정성, 스케일링 문제 등은 향후 연구가 필요한 과제로 남는다。
