# Memorizing Normality to Detect Anomaly: Memory-augmented Deep Autoencoder for Unsupervised Anomaly Detection

- **저자**: Dong Gong, Lingqiao Liu, Vuong Le, Budhaditya Saha, Moussa Reda Mansour, Svetha Venkatesh, Anton van den Hengel
- **발표연도**: 2019
- **arXiv**: https://arxiv.org/abs/1904.02639

## 1. 논문 개요

### 논문의 목표

본 논문의 목표는 딥 오토인코더(Deep Autoencoder) 기반 이상치 탐지에서 오토인코더가 비정상 입력까지 잘 재구성하는 "과적합(over-generalization)" 문제를 해결하기 위해, 메모리 모듈을 갖춘 Memory-augmented Autoencoder(MemAE)를 제안하는 것이다. 특히 정상 데이터의 대표적인 패턴(prototypical normal patterns)을 메모리에 저장하여 재구성에 활용함으로써, 비정상 데이터에서는 큰 재구성 오차를 유도하는 것을 목적으로 한다.

### 연구 문제

딥 오토인코더는 고차원 데이터를 모델링하는 강력한 도구로, 이상치 탐지에서 정상 데이터로 훈련하여 재구성 오차를 이상치 판별 기준으로 사용한다. 그러나 이 가정은 항상 유효하지 않으며, 오토인코더가 비정상 입력을 잘 재구성하는 "과적합" 현상이 관찰되었다. 이는 정상 데이터와 공통된 패턴(예: 이미지에서의 국소 에지)을 공유하는 이상치가 있을 때, 또는 디코더가 너무 강력할 때 발생한다. 이러한 문제는 이상치의 미탐(miss detection)으로 이어진다.

### 문제의 중요성

비디오 감시, 의료 영상 분석, 사이버 보안 등 다양한 영역에서 이상치 탐지는 중요한 과제이다. 고차원 데이터(특히 비디오)에서 정상 프로파일을 모델링하는 것은 매우 도전적인 문제이며, 정확하고 신뢰할 수 있는 이상치 탐지 방법의 개발이 필요하다.

## 2. 핵심 아이디어

### 중심 직관

오토인코더에 외부 메모리 모듈을 추가하여, 정상 데이터의 대표적인 패턴을 명시적으로 저장하고 재구성에 사용한다. 주어진 입력에 대해 인코딩을 쿼리로 사용하여 메모리에서 가장 관련된 항목을 검색한 후, 이 항목들을 집계하여 디코더에 전달한다. 이렇게 하면 비정상 입력은 정상 패턴만을 사용하여 재구성되어, 정상 샘플과는 다른 출력이 생성되어 큰 재구성 오차를 유도한다.

### 기존 접근 방식과의 차별점

기존 오토인코더 기반 방법들은 인코딩을 직접 디코더에 전달하여 재구성하지만, MemAE는 메모리 모듈을 통해 정상 패턴만을 사용하여 재구성한다. 이로 인해 비정상 샘플에 대한 재구성 오차가 강화되어 이상치 탐지 성능이 향상된다. 또한 제안된 방법은 Sparse Addressing을 통해 소수의 메모리 항목만 사용하도록 유도하여, 메모리가 정상 데이터의 대표적인 패턴을 효과적으로 기록하도록 만든다.

## 3. 상세 방법 설명

### 전체 구조

MemAE는 세 가지 주요 구성 요소로 이루어진다. 첫째, 인코더(입력 인코딩 및 쿼리 생성용), 둘째, 디코더(재구성용), 셋째, 메모리 모듈(메모리와 관련된 메모리 어드레싱 연산자 포함)로 구성된다.

입력 $x$가 주어지면, 인코더는 먼저 입력을 인코딩하여 잠재 표현 $z$를 얻는다. 이 잠재 표현을 쿼리로 사용하여 메모리 모듈이 주의 기반 어드레싱 연산자를 통해 메모리에서 가장 관련된 항목을 검색하고, 이를 디코더에 전달하여 재구성을 수행한다.

### 인코더 및 디코더

인코더 $f_e(\cdot)$는 입력을 정보적인 잠재 도메인으로 표현한다. 디코더 $f_d(\cdot)$는 검색된 메모리 항목을 입력으로 사용하여 샘플을 재구성한다. 수식으로 다음과 같이 정의된다:

$$z = f_e(x; \theta_e)$$

$$\hat{x} = f_d(\hat{z}; \theta_d)$$

테스트 시, 재구성 품질을 측정하기 위해 $\ell_2$ 놈 기반 평균 제곱 오차(MSE)를 사용한다:

$$e = \|x - \hat{x}\|\_2^2$$

이 오차가 이상치 탐지의 기준으로 사용된다.

### 메모리 모듈

#### 메모리 기반 표현

메모리는 행렬 $M \in \mathbb{R}^{N \times C}$로 정의되며, $N$개의 고정 차원 $C$의 실数値 벡터를 포함한다. 각 행 $\mathbf{m}\_i$는 메모리 항목이다. 주어진 쿼리(인코딩) $\mathbf{z} \in \mathbb{R}^C$에 대해, 메모리 네트워크는 다음과 같이 표현을 얻는다:

$$\hat{z} = \mathbf{w} M = \sum_{i=1}^{N} w_i \mathbf{m}\_i$$

여기서 $\mathbf{w}$는 가중치 벡터이고, $w_i$는 $i$번째 메모리 항목의 중요도를 나타낸다.

#### 어텐션을 통한 메모리 어드레싱

코사인 유사도로 메모리 항목과 쿼리 간의 유사도를 계산한다:

$$d(\mathbf{z}, \mathbf{m}\_i) = \frac{\mathbf{z} \mathbf{m}\_i^T}{\|\mathbf{z}\| \|\mathbf{m}\_i\|}$$

어드레싱 가중치는 다음과 같이 softmax 연산으로 계산된다:

$$w_i = \frac{\exp(d(\mathbf{z}, \mathbf{m}\_i))}{\sum_{j=1}^{N} \exp(d(\mathbf{z}, \mathbf{m}\_j))}$$

#### 스파스 어드레싱을 위한 Hard Shrinkage

비정상 샘플이 복잡한 메모리 항목 조합으로 잘 재구성되는 것을 방지하기 위해, 하드 수축 연산을 적용하여 어드레싱 가중치의 스파스성을 촉진한다:

$$\widehat{w}\_i = h(w_i; \lambda) = \begin{cases} w_i & \text{if } w_i > \lambda \\ 0 & \text{otherwise} \end{cases}$$

이는 ReLU 활성화 함수를 사용하여 미분 가능하게 구현된다:

$$\widehat{w}\_i = \frac{\max(w_i - \lambda, 0) \cdot w_i}{|w_i - \lambda| + \epsilon}$$

실제로 $\lambda$를 $[1/N, 3/N]$ 구간 내 값으로 설정하면 좋은 결과를 얻을 수 있다.

### 훈련

훈련 데이터 $\{x^t\}\_{t=1}^T$가 주어졌을 때, 재구성 오차를 최소화한다:

$$R(x^t, \hat{x}^t) = \|x^t - \hat{x}^t\|\_2^2$$

추가로 어드레싱 가중치의 스파스성을 촉진하기 위해 엔트로피 정규화를 최소화한다:

$$E(\widehat{\mathbf{w}}^t) = \sum_{i=1}^{T} -\widehat{w}\_i \cdot \log(\widehat{w}\_i)$$

전체 훈련 목적 함수는 다음과 같다:

$$\mathcal{L}(\theta_e, \theta_d, M) = \frac{1}{T} \sum_{t=1}^{T} \left( R(x^t, \hat{x}^t) + \alpha E(\widehat{\mathbf{w}}^t) \right)$$

여기서 $\alpha = 0.0002$를 사용한다.

## 4. 실험 및 결과

### 데이터셋

세 가지 다른 작업의 다섯 개 데이터셋에서 실험을 수행하였다. 이미지 이상치 탐지를 위해 MNIST와 CIFAR-10, 비디오 이상치 탐지를 위해 UCSD-Ped2, CUHK Avenue, ShanghaiTech, 그리고 사이버보안을 위해 KDDCUP99 데이터셋을 사용하였다.

### 기준선 방법

OC-SVM, KDE, VAE, PixCNN, DSEBM, AE, 그리고 MemAE-nonSpar(스파스 어드레싱 없음)과 비교하였다.

### 이미지 데이터 결과

MNIST에서 MemAE는 평균 AUC 0.9751을 달성하여 모든 비교 방법 중 최고 성능을 보였다. CIFAR-10에서는 0.6088의 AUC를 달성하였다. 메모리 모듈이 없는 AE와 비교할 때, 메모리 증강 모델이 현저히 우수한 성능을 보였다.

### 비디오 이상치 탐지 결과

UCSD-Ped2에서 MemAE는 AUC 0.941을 달성하여 AE(0.917)보다 우수한 성능을 보였다. ShanghaiTech에서는 0.712의 AUC를 달성하여 기존 방법들보다 경쟁력 있는 성능을 보였다. 특히稀疏 코딩 방법을 사용하는 TSC와 StackRNN보다 현저히 나은 결과를 달성하였다.

### 사이버보안 데이터 결과

KDDCUP99 데이터셋에서 MemAE는 OC-SVM, DCN, DSEBM, DAGMM 등 기존 방법들보다 우수한 정밀도, 재현율, F1 점수를 달성하였다. 이는 MemAE가 사이버보안領域에서도 효과적으로 적용될 수 있음을 보여준다.

### 메모리 시각화

MNIST 데이터에서 "9"를 정상 데이터로 훈련한 메모리를 시각화한 결과, 각 메모리 슬롯이 정상 샘플의 대표적인 패턴을 기록함을 확인하였다. 비정상 입력 "9"에 대해 "5"로 훈련된 MemAE는 입력과 현저히 다른 재구성 결과를 생성하여 큰 재구성 오차를 유도한다.

### 메모리 크기 강건성

UCSD-Ped2에서 메모리 크기 $N$에 대한 강건성을 분석한 결과, 충분히 큰 메모리 크기를 사용하면 MemAE가 안정적인 성능을 보였다.

### 실행 시간

NVIDIA GeForce 1080 Ti GPU에서 UCSD-Ped2 비디오당 프레임당 평균 0.0262초(38 fps)가 소요되어, 기존 최첨단 방법들과 유사하거나 더 빠른 속도를 보였다. 메모리 모듈 추가로 인한 추가 계산 시간은 프레임당 $4 \times 10^{-4}$초로 미미하다.

## 5. 강점, 한계

### 강점

본 논문의 주요 강점은 다음과 같다. 첫째, 오토인코더의 "과적합" 문제를 메모리 모듈을 통해 효과적으로 해결한다. 정상 패턴만을 메모리에 저장하여 재구성에 사용함으로써, 비정상 샘플에 대한 재구성 오차를 강화한다. 둘째, 데이터 유형에 대한 가정이 없어 다양한 작업에 범용적으로 적용 가능하다. 이미지, 비디오, 테이블 형식 데이터 모두에서 효과적인 성능을 보였다. 셋째, 스파스 어드레싱을 통해 소수의 대표적 정상 패턴만으로 효율적으로 재구성한다. 넷째, 메모리 크기에 대한 강건성이 뛰어나 다양한 데이터셋에서 안정적인 성능을 보인다.

### 한계 및 미해결 질문

본 논문에는 몇 가지 한계가 존재한다. 첫째, CIFAR-10과 같은 복잡한 이미지 데이터에서 성능이 제한적이다. 이는 데이터의 높은 복잡성과 클래스 내 큰 분산으로 인한 것으로, 추가적인 개선이 필요하다. 둘째, 매우 드문 이상치 유형에 대한 탐지 능력에 대한 분석이 부족하다. 셋째, 메모리 크기 $N$의 최적 선택에 대한 이론적 분석이 제한적이다. 넷째, 시계열 데이터에 대한 실험이 수행되지 않았다.

## 6. 결론

본 논문은 Memory-augmented Autoencoder(MemAE)를 제안하여 오토인코더 기반 이상치 탐지의 핵심 문제인 "과적합"을 해결하였다. 메모리 모듈을 통해 정상 데이터의 대표적인 패턴을 명시적으로 저장하고, 스파스 어드레싱을 통해 효율적으로 재구성에 활용한다. 이로 인해 비정상 샘플에서는 정상 패턴만을 사용하여 재구성되어 큰 오차를 유도하고, 이는 이상치 탐지의 기준으로 활용된다.

다양한 도메인(이미지, 비디오, 사이버보안)의 데이터셋에서의 실험을 통해 MemAE의 범용성과 효과를 입증하였다. 특히 메모리 모듈의 추가만으로 기존 오토인코더보다 현저히 향상된 성능을 달성하였다.

향후 연구 방향으로는 더 복잡한 이미지 데이터셋에서의 성능 개선, 시계열 데이터로의 확장, 그리고 다른 inúmer이 기법과의 결합이 있다.
